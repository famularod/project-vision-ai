# Definition Of Done

A change is done when it is implemented, checked, reviewed, and approved.

## Required

- [ ] The change matches the requested scope.
- [ ] Existing app behavior is preserved unless a behavior change was requested.
- [ ] Existing UI is preserved unless a UI change was requested.
- [ ] Existing data structures are unchanged unless explicitly required.
- [ ] Save, update, photo, history, storage, and cloud sync behavior are not accidentally changed.
- [ ] No unrelated refactors are included.
- [ ] No unnecessary dependencies are added.
- [ ] `npm run check` passes.
- [ ] The pull request checklist is complete.
- [ ] The user has reviewed and approved the pull request.

## Documentation

When a change affects workflow, setup, or developer behavior, update the relevant docs in the same PR.

## Merge

Merge only after user approval.
